## 1. 项目概述
**Auto-Batch-Rename-Volumes** 是一款用于自动批量重命名外部存储介质（如 U 盘、SD 卡、移动硬盘等）的实用工具。它可以根据用户预设的规则，在检测到新卷挂载时或通过手动触发，快速统一地修改卷标。

## 2. 目标用户
- 需要频繁格式化或准备大量存储设备的 IT 技术人员。
- 摄影师或媒体制作人员（需要管理大量 SD 卡）。
- 需要标准化外部存储命名规范的企业用户。

## 3. 核心功能
### 3.1 卷识别与监控
- **实时监控**: 能够检测到系统新插入的外部存储设备。
- **列表显示与选择**: 
    - 显示当前所有已连接的外部卷。
    - **多卷处理逻辑**: 当检测到多个卷时，用户可以输入数字、范围或 `all` 来选择特定的处理目标（例如：`1`, `1-2`, `all`）。

### 3.2 重命名规则引擎
用户可以选择以下两套规则之一进行重命名：

#### 3.2.1 自动递增数字模式
- **规则描述**: 根据用户定义的前缀和起始数字，自动为检测到的卷分配递增名称。
- **示例**: 前缀为 `CARD#`，起始为 `1`，则重命名结果为 `CARD#1`, `CARD#2`, `CARD#3` ...

#### 3.2.2 提取文件名模式
- **规则描述**: 遍历卷中的文件，识别出所有不同的文件扩展名，并为每种扩展名展示其第一个文件示例。在展示后，用户可以决定后续规则：
    1. **字符提取子模式**: 根据所选文件的扩展名，通过提取前后指定长度字符进行重命名。
    2. **手动输入子模式**: 绕过提取逻辑，直接为识别出的每个卷手动输入一个自定义卷名。
- **功能细节**:
    - **自动扫描与预览**: 
        - 程序会自动遍历**所有选中的目标卷**。
        - 针对每一个卷，程序都会列出该卷中发现的所有不同文件扩展名及其首个文件示例。
        - 这种全局预览可以帮助用户在处理多张卡（如 Sony 卡和 DJI 卡混合）时做出更精确的提取判断。
    - **决策分支**:
        - **选项 1 (字符提取)**: 
            - 用户输入扩展名。
            - 设置位置（前/后）及提取长度。
        - **选项 2 (手动自定义)**: 
            - 针对当前已扫描到的每个卷，程序逐一提示用户输入新的名称。
- **示例**: 
    1. 程序列出: 
       - `Volume A: .mp4 -> A001C002.mp4, .xml -> metadata.xml`
       - `Volume B: .mov -> DJI_001.mov`
    2. 用户选择规则 1 (字符提取)。
    3. 用户输入: `mp4`。
    4. 结果: Volume A 按规则重命名，Volume B 若无 mp4 则跳过并提示。

### 3.3 安全与过滤
- **持久化白名单**: 用户可以选取特定卷加入白名单。白名单数据存储在本地（JSON 格式），程序重启后依然生效。
    - **管理功能**: 支持查看白名单列表、从白名单中移除特定项、以及恢复白名单至空状态（重置）。
- **系统黑名单**: 自动过滤 Macintosh HD 等关键系统分区。
- **确认机制**: 执行前展示预览并要求确认。
- **全数字交互规范**: 为了避免混淆，所有菜单选项统一使用数字标识（如 `1`, `2`, `0`）。

### 3.4 历史与日志
- 记录每次成功重命名的操作（设备标识、旧名称、新名称、时间）。

## 4. 用户故事 (User Stories)
- **场景 A**: 作为一个活动摄影师，我插上 10 张 SD 卡，我希望它们能自动被重命名为 `CARD#1` 到 `CARD#10`。
- **场景 B**: 作为一个数字影像工程师，我经常会同时拷贝多张sony或者dji的存储卡，他们的默认卷名都是untitle，但是mac又无法同时挂载多个相同卷名的volume，我希望多个volume能在被重命名后同时挂载到电脑上

## 5. 技术需求
- **跨平台支持**: 优先支持 macOS (使用 `diskutil`)。
- **开发语言**: Python 3.x。
- **核心模块**:
  - `main.py`: CLI 交互与业务逻辑。
  - `diskutil`: 系统原生命令，用于卷重命名及详细信息获取。
  - `os.walk`: 用于遍历卷内文件以提取名称。
  - **检测逻辑**: 通过扫描 `/Volumes` 目录并结合 `diskutil info` 匹配 `External`、`USB` 等关键字来精准识别外部存储设备。
- **验证**: 通过 `test_mock.py` 进行单元测试验证重命名规则。

## 6. 非功能性需求
- **性能**: 扫描和重命名操作应在数秒内完成。
- **可靠性**: 重命名失败时应提供明确的错误信息（如权限不足、磁盘被占用）。
- **易用性**: 配置文件应简单直观。

## 7. 风险评估
- **系统权限**: 修改卷标通常需要管理员权限。
- **文件系统限制**: 不同文件系统（APFS, ExFAT, FAT32）对卷标长度和字符有不同限制。

## 8. 路线图 (Roadmap)

- **V1.1**: 增加 CSV 映射表支持。
